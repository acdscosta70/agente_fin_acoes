# ia_agente_fin_acoes
Agentes de IA no mercado de Ações
Analista Fundamentalista - Analise Funcional
Analista Tecnico - Analise Tecnica
• Arquitetura da Solução:
Implementei uma arquitetura de orquestração multi-agente operando em um pipeline de processamento sequencial. A inteligência do sistema foi dividida entre dois agentes autônomos com personas e papéis distintos: um Analista Financeiro Fundamentalista e um Analista Técnico de Ações. O agente fundamentalista realiza a coleta e interpretação de balanços, demonstrações de resultados e métricas de rentabilidade. Na sequência, o agente técnico consome dados de mercado em tempo real para avaliar padrões gráficos e indicadores algorítmicos (como RSI e MACD) visando identificar pontos de entrada e saída. Ambos os agentes atuam com autonomia para acessar a web, compondo um fluxo RAG (Retrieval-Augmented Generation) dinâmico.• Tecnologias Utilizadas:O core da aplicação foi desenvolvido em Python utilizando CrewAI para a orquestração estruturada dos agentes e suas respectivas tarefas. O framework LangChain foi utilizado como camada de abstração para integração com o motor de inferência, operando com o modelo GPT-4o através da API da OpenAI. Para a capacidade de tool use (acesso externo em tempo real), integramos a SerperDevTool gerenciada via chaves de API restritas no ambiente (.env).• Como foi feito o controle de custo:
O controle de custos foi otimizado diretamente na arquitetura dos prompts e no acionamento de ferramentas. A economia de tokens no modelo GPT-4o foi garantida restringindo os agentes a formularem buscas altamente específicas em formato JSON (ex: {"search_query": "BBAS3 balanços financeiros 2023"}), reduzindo o volume de texto processado nas respostas de busca. Adicionalmente, a limitação de delegação (allow_delegation=False) no agente fundamentalista evitou loops infinitos de delegação de tarefas que consomem redundante de tokens.• Como foi feita a observabilidade da aplicação:
A observabilidade foi implementada monitorando ativamente o paradigma ReAct (Reason and Act) dos LLMs. O sistema foi configurado com injeção detalhada de logs no orquestrador (verbose=2) e nível de debug nos agentes (verbose=True). Isso permitiu o rastreamento linear de toda a execução em produção: monitorando a intenção do agente ("Thought"), a chamada de função exata ("Action: Search the internet"), o payload enviado ("Action Input") e o retorno da API de terceiros ("Search results"). Esses logs contínuos garantem que possamos auditar alucinações de dados financeiros e validar o tempo de resposta das integrações.
